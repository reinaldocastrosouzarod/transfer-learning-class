Transfer Learning

Objetivo da aula (30 min)

  • Compreender o conceito de Transfer Learning.
  • Diferenciar fine‑tuning e feature‑extraction.
  • Aplicar o pipeline em um caso prático (CIFAR‑10).

Definição formal

Transfer Learning = reutilização de representações aprendidas por um modelo pré‑treinado para resolver uma nova tarefa com menos dados rotulados.

— Wang & Chen (2023), Introduction to Transfer Learning, §1.2

Motivações pedagógicas (ICD‑00)

  • Reduz custo computacional – menos épocas, menos parâmetros a atualizar.
  • Menor necessidade de dados rotulados – essencial em cursos iniciais de IA.
  • Melhoria de performance quando o domínio da nova tarefa é semelhante ao domínio original (ex.: imagens).

Estratégias de Transfer Learning

  1. Feature‑Extraction: congela todas as camadas exceto a última.
  2. Fine‑Tuning: descongela parte (ou todas) as camadas e continua o treinamento.
  3. Adapter‑Based (avançado): insere módulos leves entre camadas.

Tabela comparativa (Wang & Chen, 2023, Table 2) – adaptável ao nível de graduação.

Pipeline típico

  1. Selecionar modelo pré‑treinado (ex.: ResNet‑18, VGG‑16, BERT).
  2. Adaptar a camada de saída ao número de classes da tarefa alvo.
  3. Congelar ou descongelar camadas conforme a estratégia.
  4. Treinar com o dataset alvo (geralmente < 10 épocas).
  5. Validar e analisar métricas (acurácia, top‑k, matriz de confusão).

Ilustração inspirada em ICD‑04 – Classificação + Métricas.

Estudo de caso – CIFAR‑10

  • Dataset: 60 k imagens 32×32, 10 classes.
  • Modelo base: ResNet‑18 pré‑treinado em ImageNet.
  • Estrategia: feature‑extraction (congelar todas as conv‑layers).
  • Hiper‑parâmetros sugeridos:
    • Batch = 128
    • Learning‑rate = 0.01, momentum = 0.9
    • Épocas = 5 (≈ 8 min em CPU típica de laboratório)
  • Objetivo: demonstrar que a acurácia já ultrapassa 70 % sem fine‑tuning.

Trecho de código (PyTorch)

import torch, torch.nn as nn, torch.optim as optim
from torchvision import datasets, transforms, models

# 1️⃣ Transformações e carga dos dados
transform = transforms.Compose([
    transforms.Resize(224),
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
])
train = datasets.CIFAR10('data', train=True, download=True, transform=transform)
val   = datasets.CIFAR10('data', train=False, download=True, transform=transform)
train_loader = torch.utils.data.DataLoader(train, batch_size=128, shuffle=True, num_workers=2)
val_loader   = torch.utils.data.DataLoader(val,   batch_size=128, shuffle=False, num_workers=2)

# 2️⃣ Modelo pré‑treinado + ajuste da camada final
model = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1)
for p in model.parameters():
    p.requires_grad = False            # feature‑extraction
model.fc = nn.Linear(model.fc.in_features, 10)   # 10 classes
model = model.to('cuda' if torch.cuda.is_available() else 'cpu')

criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(model.fc.parameters(), lr=0.01, momentum=0.9)

Para fine‑tuning, remova o bloqueio requires_grad = False nas últimas n camadas.

Métricas recomendadas (ICD‑04)

  • Acurácia Top‑1 – proporção de previsões corretas.
  • Acurácia Top‑5 – útil quando há classes visualmente semelhantes.
  • Matriz de Confusão – analisa erros por classe.
  • Curvas de loss e learning rate ao longo das épocas.

Discussão em turma

  • Quando o fine‑tuning pode ser mais vantajoso que feature‑extraction?
  • Quais são os riscos de aplicar um modelo pré‑treinado em domínios muito diferentes (ex.: imagens médicas vs. natural)?
  • Como escolher o modelo base considerando tamanho, profundidade e disponibilidade de pesos?
