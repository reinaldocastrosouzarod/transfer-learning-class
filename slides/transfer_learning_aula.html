Transfer Learning — ENG4502 Contextualização
Conceito

O que é Transfer Learning?

"Transfer Learning consiste em aproveitar o conhecimento adquirido por um modelo em uma tarefa fonte (source) para melhorar o aprendizado em uma tarefa alvo (target), especialmente quando os dados rotulados da tarefa alvo são escassos."

— Wang & Chen (2023), Introduction to Transfer Learning: Algorithms and Practice, §1.2

🎯 Tarefa Fonte (Source)

Modelo treinado em grande base de dados. Ex.: ResNet-18 treinado em ImageNet (1,2 milhão de imagens, 1000 classes).

🎯 Tarefa Alvo (Target)

Problema que queremos resolver com poucos dados. Ex.: classificar imagens de CIFAR-10 (10 classes, 50k imagens).

Transfer Learning — ENG4502 Definição
Intuição

Por que funciona? Uma analogia

Você já aprendeu a andar de bicicleta. Quando você tenta andar de moto pela primeira vez, não começa do zero — você já sabe o que é equilíbrio, como virar, como travar.

Em visão computacional funciona igual: um modelo que aprendeu a detectar bordas, texturas e formas em ImageNet já sabe "ver". Basta ensinar o que são as suas novas classes.

💡 As camadas convolucionais iniciais aprendem detectores genéricos (bordas, cores, gradientes) que são úteis em qualquer tarefa de visão.
🚲

Bicicleta = ImageNet

Habilidade genérica aprendida com muitos dados e esforço.

🏍️

Moto = CIFAR-10

Nova tarefa aprendida rapidamente aproveitando o conhecimento anterior.

Estratégias

Feature Extraction vs. Fine-Tuning

🧊 Feature Extraction
  • ✅ Congela todas as camadas do backbone
  • ✅ Treina apenas a camada final (classificador)
  • ✅ Muito rápido — poucos parâmetros para atualizar
  • ✅ Ideal quando há poucos dados na tarefa alvo
  • ⚠️ Menos flexível — features fixas do domínio fonte
🔥 Fine-Tuning
  • ✅ Descongela parte (ou todas) as camadas
  • ✅ Adapta as features ao domínio alvo
  • ✅ Melhor performance quando há dados suficientes
  • ⚠️ Mais lento — muitos parâmetros para atualizar
  • ⚠️ Risco de overfitting com datasets pequenos
📌 Na prática de hoje usaremos Feature Extraction — mais simples, mais rápido, e já demonstra ganhos expressivos de acurácia.

Pipeline

Do dado ao modelo: passo a passo

  • 1
    Dados — Carregar CIFAR-10 com torchvision.datasets.CIFAR10. Aplicar transforms: resize para 224×224 (input do ResNet), normalização.
  • 2
    Modelo — Instanciar ResNet-18 com pesos do ImageNet (ResNet18_Weights.IMAGENET1K_V1). Congelar backbone, trocar camada final.
  • 3
    Treinamento — Usar SGD (lr=0.01, momentum=0.9) apenas sobre os parâmetros da camada final. Loop de 5 épocas.
  • 4
    Avaliação — Calcular acurácia no conjunto de validação a cada época. Plotar curva de loss. Gerar matriz de confusão.
  • 5
    Análise — Comparar resultado com treino do zero. Discutir: o que muda se descongelarmos algumas camadas?

Código

Implementação em PyTorch

import torch, torch.nn as nn, torch.optim as optim
from torchvision import datasets, transforms, models

# ── 1. Transformações ────────────────────────────────────────
transform = transforms.Compose([
    transforms.Resize(224),                      # ResNet espera 224×224
    transforms.ToTensor(),
    transforms.Normalize((0.5,0.5,0.5), (0.5,0.5,0.5))
])

# ── 2. Dataset e DataLoader ──────────────────────────────────
train_ds = datasets.CIFAR10('data', train=True,  download=True, transform=transform)
val_ds   = datasets.CIFAR10('data', train=False, download=True, transform=transform)
train_loader = torch.utils.data.DataLoader(train_ds, batch_size=128, shuffle=True)
val_loader   = torch.utils.data.DataLoader(val_ds,   batch_size=128)

# ── 3. Modelo pré-treinado + adaptação ──────────────────────
model = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1)
for p in model.parameters():
    p.requires_grad = False            # ← congela backbone (feature extraction)
model.fc = nn.Linear(model.fc.in_features, 10)  # ← 10 classes CIFAR-10

# ── 4. Treinamento ───────────────────────────────────────────
optimizer = optim.SGD(model.fc.parameters(), lr=0.01, momentum=0.9)
criterion = nn.CrossEntropyLoss()

Cuidados

Limitações e riscos do Transfer Learning

⚠️ Negative Transfer

Quando o domínio fonte é muito diferente do alvo, o modelo pré-treinado pode prejudicar o aprendizado. Ex.: usar ResNet (imagens naturais) para diagnóstico por raio-X sem fine-tuning cuidadoso.

⚠️ Domain Shift

A distribuição dos dados fonte e alvo é diferente. As features aprendidas podem não generalizar bem para o novo domínio.

📏 Tamanho do Modelo

Modelos grandes como ResNet-50, VGG-16 ou ViT podem ser pesados demais para deploy em dispositivos embarcados ou com restrições de memória.

⚖️ Viés do Modelo

O modelo pré-treinado pode carregar vieses do dataset fonte (ex.: ImageNet tem sub-representação de grupos demográficos). Importante avaliar equidade.
